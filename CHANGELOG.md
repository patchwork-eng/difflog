# Changelog

## v1.0.1 — 2026-03-21

### ✨ Features
- A new landing page for difflog.io has been added, enhancing the overall user experience.
- Introduced a Cloudflare Worker for license validation, which is now integrated into the Action for real-time validation.
- Marketplace branding has been updated along with a pagination fix, and Sentry has been integrated for better error tracking.
- Added a subscription page that is now fully functional with Stripe payment links wired to pricing cards.
- Implemented usage logging to KV on successful validation and created a stub for the Resend email hook during checkout.
- A new `/demo` endpoint has been added, featuring a live GitHub repository demo UI.
- The footer on changelogs now includes the Difflog branding, and an interactive demo has been added to the landing page.
- Enhanced the demo output with smarter formatting, commit hashes, and a conversion hook for improved shareability.

### 🐛 Bug Fixes
- Improved error handling throughout the application and added a license skeleton to enhance maintainability.
- Fixed the internal email address across all pages, replacing it with hello@difflog.io.
- Updated the Worker validation URL to point to api.difflog.io, ensuring correct functionality.
- Added a "Get started free" call-to-action to the Free pricing card to improve user engagement.
- Included a `.nojekyll` file to disable Jekyll for GitHub Pages, preventing potential deployment issues.
- Switched to GitHub Actions for Pages deployment, streamlining the deployment process.

### 🔧 Chores & Maintenance
- The README file has been updated to include comprehensive usage instructions, improving onboarding for new users.
- Documentation has been expanded to include a privacy policy, terms of service, and a refund policy, ensuring compliance and transparency.
- A Jest test suite has been added, along with 142 new corner case tests for both the Action and Worker, enhancing test coverage and reliability.
- Various updates were made to the Worker configuration, including the real KV ID and the api.difflog.io route, to ensure proper functionality.
